# huang bai

## 核心目标

根据用户文案直接生成 3 张 3:4 竖版短视频封面。封面要有强点击感、强层级、强质感，不做普通干净海报。

默认人脸参考图：`assets/face/default-face.png`。生成人物时必须把这张图片文件作为真实图片输入传给生图模型，人物长相和发型必须复刻原图；表情默认优化为自信的笑容，除非用户明确要求复刻原图表情。

## 硬性执行约束

- 禁止只在提示词里写 `assets/face/default-face.png` 或本地路径来声称“参考了人脸”。路径文字不会自动把图片传给模型。
- 必须使用支持图片输入的生图、编辑或合成流程，把 `assets/face/default-face.png` 作为输入图传给模型。
- 如果当前可用工具只能纯文本生图，不能传入图片文件，则不要生成封面；必须先告知用户无法满足“固定人脸参考”的硬约束，并要求用户上传人脸图或启用可传图的本地/API流程。
- 如果使用本地 `imagegen` 技能/CLI，优先走 `edit` 流程，并传入 `--image <default-face.png>` 与 `--input-fidelity high`。如果一次生成多图不稳定，逐张生成。
- 生成结果必须优先检查脸是否像参考图；如果长相或发型明显偏离，应重新生成，不要把偏离结果当成完成。表情应是自信、自然、有亲和力的笑容，不能变成陌生人、夸张大笑、僵硬假笑或严肃冷脸。

成功封面样例：

- `assets/style-references/web-design-cover.png`
- `assets/style-references/world-model-cover.png`
- `assets/style-references/ai-short-drama-cover.png`

生成前先参考这些样例的版式质感、标题层级、人物道具关系和背景世界观，但不要照抄具体文字、构图或题材。

## 工作流

1. 把文案当作灵感来源，不要当作画面清单。
2. 内部先确定一个最适合的主标题关键词；默认 3 张封面使用同一个主标题和同一个副标题。
3. 先做创意推演：主题是什么、核心钩子是什么、用什么道具/场景能把钩子讲清楚、人物动作和道具之间有什么逻辑。
4. 每张封面只表达一个最强钩子，避免塞满步骤、卖点和说明文字。
5. 生成 3 张本质不同的封面，差异体现在构图、视觉重心、表达方式上，不靠换标题制造差异。
6. 为了稳定复刻本人长相和发型，必须使用支持参考图编辑/合成的方式，把人脸图片作为图片输入传给模型；不要只靠文生图提示词重新生成一张脸。表情默认改为自信的笑容。
7. 除非用户要求，否则不要输出分析、标题方案或提示词草稿，直接生成图片。

## 成功封面公式

复用之前成功图的共同特征：

- 大标题压顶：标题占顶部 25% 到 40% 空间，字体厚重、立体、有描边、投影和纹理。标题质感优先参考成功样例里的粗粝海报印刷、招牌字、贴纸字、搪瓷/亚克力字、手绘涂刷字、复古包装字，不默认使用金属字。
- 人物是视觉锚点：人物位于中间或偏下，脸清楚，和前景道具发生互动，而不是站着摆拍。
- 前景道具要夸张：手机、电脑、屏幕、道具、产品、角色物件可以明显前置、放大、带透视，形成“伸到镜头前”的冲击。
- 背景有世界观：背景不是纯色棚拍，而是和主题相关的场景世界，例如复古街区、电影棚、游戏空间、工具工作台、产品宇宙、夸张实景。
- 质感靠设计逻辑、真实材料和画面层级，不靠金属质感堆叠或科技感光效。可以使用真实影棚光、招牌灯、环境灯、纸张颗粒、胶片磨损、油墨纹理、道具材质、景深、烟雾、反光地面来增加质感。
- 信息有主次：可以有少量无文字符号，例如对勾、叉号、箭头、圈注，但必须服务主钩子，不能平均铺满。
- 画面是“可控高密度”：不是极简，也不是乱堆；标题最大，人物第二，道具第三，背景最后。

## 参考图风格总结

- 标题更像“真实商业海报标题”，不是 UI 里的普通字体，也不是廉价金属特效。可以是粗白纸面大字、复古招牌字、彩色贴纸字、搪瓷糖果字、斜切海报字、油墨颗粒字、旧海报磨损字。
- 标题允许有厚度、描边和投影，但厚度要服务可读性和冲击力，不要整张都做成金属、铬合金、黄金、银色镭射。
- 光效来自场景里的真实灯光：街景招牌、摄影棚灯、暖色舞台灯、屏幕反光、环境边缘光。不要用科技感激光、蓝紫光束、赛博霓虹、粒子爆炸、能量环、HUD 光效。
- 色彩要根据题材建立世界观，可以是复古街景、电影棚、设计工作台、游戏厅、产品拍摄棚；不要默认蓝紫科技风，也不要默认金色金属风。
- 标题、人物、道具、背景要像同一张设计完成度高的商业封面，不能像把 3D 字、科技光、UI 卡片随意叠在一起。

## 创意逻辑

生成前先让模型自己想创意，不要机械套模板。每个元素都要能回答“为什么在这里”：

- 人物动作要解释核心钩子，例如展示、对比、发现、操控、拿出结果。
- 核心道具要直接代表内容主题，例如手机代表可操作性，电脑代表网页结果，导演监视器代表短剧，玩具马代表短剧奇观。
- 背景要承接主题世界，例如短剧用片场/剧院/电影街景，网站用设计工作台/浏览器空间，游戏用街机/游戏厅。
- 箭头、对勾、叉号、圈注只能帮助理解，不要变成装饰噪音。
- 不要为了“高级”默认使用金属字、蓝紫色、科幻舱、赛博霓虹、过量发光、粒子、速度线；这些容易产生“AI 味”和廉价科技感。

## 标题规则

- 主标题必须是短关键词、名词或名词短语。
- 主标题尽量 5 个字以内。
- 默认 3 张封面使用同一个主标题。只有当用户明确要求多标题测试，或文案包含明显不同选题方向时，才生成不同标题。
- 副标题也默认保持一致，不要为了多方案强行变化。
- 主标题位于顶部，占满顶部主要空间。
- 字体要大、加粗、有设计感，不要像普通输入文字；优先使用粗白纸面字、复古招牌字、彩色贴纸字、搪瓷/亚克力字、手绘涂刷字、斜切海报字、油墨颗粒字、旧海报磨损字。
- 可以少量使用立体厚度、描边和投影，但不要默认金属质感；除非题材强相关，否则禁止整张标题做成铬合金、黄金、银色镭射或廉价金属字。
- 不要用动作短语或流程短语做主标题，例如“三步上线”“40秒搞定”“不用代码”“一键高级”。
- 副标题最多 1 条，能不用就不用；总文字越少越好。

## 人物规则

- 必须使用 `assets/face/default-face.png` 作为人物长相参考，并把该图片文件作为图片输入传给模型。
- 禁止仅在提示词中描述“参考 assets/face/default-face.png”；这不算使用参考图。
- 人物长相和发型必须与参考图一致。
- 表情默认是自信的笑容：嘴角自然上扬，眼神稳定，有亲和力和专业感；不要严肃冷脸、夸张大笑、网红式假笑、挤眉弄眼或明显改变五官。
- 动作只发生在身体、手和道具上，服装可以根据主题更换。
- 人物可以是大头、半身或局部近景，脸要清楚。
- 可以在人物周围添加极少量与主题相关的无文字说明性元素，例如对勾、叉号、箭头、圈注，但必须有主次。
- 通过动作描述引导人物，但动作只写身体、手和道具，例如：手指向屏幕、拿着手机、做演示动作、托举道具。
- 人物最好和核心道具发生互动，例如双手托举、把手机递向镜头、指向结果、骑乘/抓握主题物件、拿着关键道具。

## 背景规则

- 背景必须与主题相关，单独设计。
- 不要使用参考图原背景。
- 不要使用纯黑或纯白背景。
- 背景要有明确世界观和纵深，但不能抢标题和人物。
- 整体保持高级感、商业感、冲击感和聚焦感。

## 画面规则

- 优先结构：大标题 + 人物 + 一个核心视觉物 + 主题世界背景。
- 核心视觉物必须服务钩子，例如产品、界面、影视画面、工具面板、结果对比。
- 可以使用箭头、圈注、卡片、边框磨损、胶片颗粒、实物质感、真实灯光，但必须围绕主钩子。
- 避免平均用力、元素无主次、过多 UI 卡片、复杂流程图、假 logo、水印、二维码、过多小字、过多标签。
- 避免金属质感堆叠、蓝紫科幻光、赛博霓虹、科技激光、HUD 光效、过度发光、粒子速度线堆叠、抽象 AI 背景。

## 二期占位

未来能力：上传视频后自动解析或推断脚本，生成封面文案，再生成封面。当前 `huang-bai` 只支持粘贴文案生成封面；如果用户要求视频自动生成封面，需要说明二期能力尚未实现，但可以在用户提供或确认文案后继续生成封面。
